API Documentation

Welcome to the API Marketplace documentation. Learn how to list your APIs or integrate third-party APIs into your applications.

Introduction

API Marketplace is a platform that allows developers to list, manage, and monetize their APIs. Consumers can discover and subscribe to APIs with flexible pricing plans.

Base URL: https://api.marketplace.com/v1

Authentication

All API requests require authentication using an API key. Include your API key in the request header:

curl -X GET "https://api.marketplace.com/v1/data" \ -H "X-API-Key: your_api_key_here"
Security: Never share your API key publicly or commit it to version control.

Getting Your API Key

1. Sign up for an account
2. Navigate to your dashboard
3. Click "Generate API Key"
4. Copy and securely store your key

Rate Limits

Rate limits are enforced based on your subscription plan:

Plan Requests/Month Rate Limit
Free 1,000 10 req/minute
Basic 50,000 100 req/minute
Pro 500,000 1,000 req/minute
Enterprise Unlimited Custom

Listing Your API

API providers can list their existing APIs on the platform. Follow these steps:

POST /apis

Create a new API listing on the platform.

Request Body

Parameter Type Description
name required string Name of your API
description required string Brief description of your API
base_url required string Base URL of your API
category optional string API category (e.g., weather, finance)
pricing_model required string free, subscription, or pay-per-use

Example Request

{ "name": "Weather Data API", "description": "Real-time weather information", "base_url": "https://api.weather.com/v1", "category": "weather", "pricing_model": "subscription" }

Example Response

{ "id": "api_abc123", "name": "Weather Data API", "status": "active", "created_at": "2024-01-15T10:30:00Z", "api_key": "sk_live_abc123xyz" }

Making Requests

API consumers can access listed APIs using their subscription API key.

GET /proxy/{api_id}/endpoint

Make a proxied request to a listed API.

Headers

Header Value Description
X-API-Key string Your API Marketplace key
Content-Type application/json Request content type

Example Request

curl -X GET "https://api.marketplace.com/v1/proxy/api_abc123/weather?city=London" \ -H "X-API-Key: your_api_key_here" \ -H "Content-Type: application/json"

Example Response

{ "temperature": 15.5, "condition": "Cloudy", "humidity": 75, "wind_speed": 12 }

Handling Errors

The API uses standard HTTP response codes to indicate success or failure.

Code Status Description
200 OK Request successful
400 Bad Request Invalid request parameters
401 Unauthorized Invalid or missing API key
403 Forbidden API key suspended or expired
429 Too Many Requests Rate limit exceeded
500 Internal Server Error Server error

Error Response Format

{ "error": "Rate limit exceeded", "code": 429, "limit": 100, "period": "minute", "reset_at": "2024-01-15T11:00:00Z" }
